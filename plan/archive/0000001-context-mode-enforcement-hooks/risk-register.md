## Risks

| ID | Category | Description | Likelihood | Impact | Mitigation | Status |
|----|----------|-------------|------------|--------|------------|--------|
| R-001 | technical | Allowlist too narrow — a legitimate Bash command (e.g. `npm run build`, `pnpm install`) is blocked because it pipes through `grep` or contains a blocked token | medium | high | Allowlist is explicit and documented in REQ-002; easy to extend. Monitor for false-positive blocks during validate phase. | open |
| R-002 | technical | Claude Code `settings.json` hook schema differs from the assumed format — `matcher`, `type`, or `command` field names are wrong | medium | high | Verify exact schema from Claude Code docs before codegen (Q-001 in execution plan). Add a smoke test that installs and invokes a no-op hook. | open |
| R-003 | technical | Hook scripts receive tool name only on stdin, not the full tool input JSON — `command` field unavailable for Bash pattern matching | medium | high | Verify Claude Code hook stdin payload format (Q-002 in execution plan). If only tool name is available, Bash pattern blocking is impossible; fallback is blocking all Bash unconditionally (less desirable). | open |
| R-004 | technical | Bash shebang (`#!/usr/bin/env bash`) does not resolve on Windows — Claude Code invokes the hook and gets a non-zero exit or parse error | medium | medium | Test on Windows during validate phase. If bash is unavailable, provide `.ps1` equivalents or an inline command wrapper. Tracked as a known risk in design.md. | open |
| R-005 | technical | `settings.json` merge logic in setup overwrites existing hook entries — user's existing hooks are lost | low | high | Implement additive merge (read → merge → write) rather than replace. Test with a pre-existing `settings.json` that already has hook entries. | open |
| R-006 | operational | Agent receives block message but does not understand the `ctx_*` redirect — calls the tool again instead of switching | low | medium | Block message must be specific and imperative, not advisory. Include the exact replacement call syntax. This is a quality-of-block-message concern, not a hook correctness concern. | open |
| R-007 | operational | Hook scripts drift from upstream `mksglu/context-mode` after the upstream contribution — two versions diverge | low | low | Scripts are versioned in planifest-framework. Upstream sync is a known follow-up tracked in the roadmap. Impact is low until divergence causes a functional difference. | open |
