# Scope - context-mode Enforcement Hooks

## In Scope

- Bash hook script that blocks the `Grep` tool unconditionally and redirects to `ctx_execute(language:"shell", code:"grep ...")`
- Bash hook script that blocks the `Bash` tool when the command matches `grep`, `rg`, `curl`, `wget`, or inline HTTP patterns, with an explicit allowlist for `git`, `mkdir`, `rm`, `mv`, `cd`, `ls`, `npm install`, `pip install`
- Bash hook script that blocks the `WebFetch` tool unconditionally and redirects to `ctx_fetch_and_index` + `ctx_search`
- Block messages that name the specific `ctx_*` replacement tool — not a generic error message
- Hook scripts stored in `planifest-framework/hooks/context-mode/` (source) and installed to `.claude/hooks/context-mode/` (target)
- `setup.sh` + `setup.ps1` changes: copy scripts and write `PreToolUse` hook entries into `.claude/settings.json` when `--context-mode-mcp` is passed
- `setup/claude-code.sh` + `setup/claude-code.ps1` tool config declarations for hook paths
- Claude Code only

---

## Out of Scope

- Blocking the `Read` tool — intent is indistinguishable at hook level (editing vs analysing); routing rules remain the mechanism for Read discipline
- Blocking `Edit` or `Write` tools — these are always correct operations; no blocking required
- Hook support for Cursor, Windsurf, Cline, Antigravity — deferred; hook architectures not confirmed for these tools
- Removal of installed hooks when `--context-mode-mcp` flag is absent on re-run — no uninstall logic in v1
- Runtime detection of whether context-mode MCP server is actually running — the flag is an explicit opt-in; if the user passes it, the server is assumed configured
- Configurable allowlists per project — allowlist is hardcoded in the script for v1
- The upstream contribution of hook scripts to `mksglu/context-mode` — tracked as a follow-up after this pipeline run

---

## Deferred

- **Hook support for other tools (Cursor, Windsurf, Cline, Antigravity):** Blocked until each tool's hook architecture is confirmed. No agent tool blocks are deferred — only the per-tool wiring.
- **Output-size estimation for Bash:** Blocking based on predicted output volume rather than command pattern match. Too complex for v1; pattern-based blocking is the v1 approach.
- **Configurable allowlist:** Per-project override of the blocked/allowed command patterns. Requires a config schema design. Blocked until the hook interface is stable.
- **Hook uninstall:** Re-running setup without `--context-mode-mcp` does not remove previously installed hooks. An uninstall path requires tracking installed state. Deferred.
- **Upstream contribution to `mksglu/context-mode`:** Scripts are authored in planifest-framework. Contribution happens after this pipeline run — tracked in the roadmap.
