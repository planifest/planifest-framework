# Requirement: REQ-011 - skill-registries

**Feature:** 0000005-framework-governance
**Source:** plan-scoped and permanent skill registry user stories (feature-brief.md)
**Priority:** must-have

---

## Functional Requirements

- `planifest-framework/external-skills.json` is the permanent registry; setup.ps1/setup.sh generates it by scanning `planifest-overrides/capability-skills/` if that directory exists; if `planifest-overrides/` is absent, setup writes `{}` and proceeds without error
- `plan/current/external-skills.json` is the plan-scoped registry; written by the orchestrator when a one-time skill is classified; absent when no one-time skills exist
- All phase agents must check `plan/current/external-skills.json` (plan-scoped) first, then `planifest-framework/external-skills.json` (permanent) at phase start
- The orchestrator at each phase transition merges both registries and surfaces only skills relevant to the upcoming phase and declared stack — not all skills indiscriminately
- planifest-ship-agent must include `plan/current/capability-skills/` and `plan/current/external-skills.json` in the plan archive at P7 so one-time skills are preserved with the run that used them

## Acceptance Criteria

- [ ] `planifest-framework/external-skills.json` exists (initially `{}`) and is regenerated by setup
- [ ] Orchestrator writes `plan/current/external-skills.json` when a one-time skill is processed
- [ ] Phase agents check plan-scoped registry first, permanent second
- [ ] Ship-agent archives plan-scoped skills and registry at P7

## Dependencies

- REQ-010 (capability-skill-intake) — intake populates registries
