## Before Merging

### REC-001 — Fix skill-sync.sh security findings (P5 F-001/002/003)

Three security findings in `planifest-framework/scripts/skill-sync.sh` should be addressed before production use:

1. **F-001** (`get_skill_scope` L78): Replace `$name` interpolation in `node -e` string with env-var pattern.
2. **F-002** (all `cmd_*` functions): Add `validate_skill_name()` — `^[a-zA-Z0-9_-]+$` — before any path operation.
3. **F-003** (`_fetch_skill` L218): Validate `--from` URL starts with `https://`.

These are all in `skill-sync.sh` and can be addressed in a single focused change-agent pass.

### REC-002 — Add `setup.ps1` skill subcommand routing

`setup.sh add-skill` delegates to `skill-sync.sh`. The PowerShell equivalent `setup.ps1` does not yet route `add-skill`/`remove-skill`/`preserve-skill`/`unpreserve-skill` to `skill-sync.ps1`. Windows users must call `skill-sync.ps1` directly, which is inconsistent. Add equivalent routing at the top of `setup.ps1`.

## Near-term Improvements

### REC-003 — Add `test-skill-sync.sh` automated test suite

`skill-sync.sh` has no automated tests. The operations (add, install, remove, sync, preserve, unpreserve) and the `validate_skill_name` guard (once added) should all have test coverage. Use a mock `curl` wrapper to avoid real network calls.

### REC-004 — Fix `test-setup-telemetry.sh` for Windows

`make_workspace()` creates temp git repos that fail on Windows due to "dubious ownership". Refactor to use the actual project root with `.claude/` backup/restore, running tests outside the ctx_execute sandbox. Currently 9/11 tests blocked (TD-004).

### REC-005 — Commit message subject line length in advisory hook

The advisory hook's `${#SUBJECT}` counts bytes, not visible characters. Commit subjects with multi-byte characters (em-dashes, non-ASCII) show inflated lengths. Consider using `echo -n "$SUBJECT" | wc -m` for character count.

### REC-006 — Skill content validation on fetch

`_fetch_skill()` in `skill-sync.sh` fetches a `SKILL.md` from GitHub raw content without validating the response. Add a minimal check: (1) max file size (e.g., 512 KB), (2) confirm file starts with `---` (YAML frontmatter), (3) confirm `name:` field present. This reduces supply-chain risk if a skill is tampered with.

## Future Iterations

### REC-007 — Skill version pinning

Currently skills are always fetched from the `main` branch of the Anthropic skills repo. Add support for pinning to a git tag or commit SHA in `external-skills.json`, allowing reproducible installs and controlled upgrades.

### REC-008 — `setup.sh` dry-run mode

Add `--dry-run` flag to `setup.sh` that prints what would be installed/changed without making any filesystem modifications. Useful for CI validation that setup would succeed.
